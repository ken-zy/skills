> **2026-05-20 新增观察**（基于《味对了》）：
> - **城市现场第一感可直接成文**：从“落地深圳，还没走出航站楼”这种具体空间开头，靠机场 AI 广告、凌晨大排档、外卖电动车这些现场信号撑住判断，不需要先铺背景。
> - **第一感后必须自我反驳一轮**：承认“AI 广告多，不等于深圳真做出 AI”“大排档热闹，不等于这里的人在干活”，再落到“脑子冷一下，还成立”。这让直觉判断不显得轻飘。
> - **味道/节奏这类抽象词要被场景钉住**：抽象判断前后必须有可见物件和动作，比如墙上广告、烧鹅饭、旁桌讨论、电动车滑过门口。
> - **短标题可以只命名一种体感**：“味对了”不是事件摘要，是作者对场域的判断。适合旅行、参会、城市、线下圈子这类素材。
>
> **2026-05-18 新增观察**（基于《AI 从工具变成 Agent 后，人类到底还控制多少》）：
> - **加粗骨架金句继续走密**：长文里 2-3 处加粗判断式承担板块锚点功能（**AI 不需要有自己的意志，也能改变未来方向。** / **AI 未来方向短期由人类决定，长期由反馈系统决定。** / **AI 不需要掌握方向盘。它只要改变路的坡度。**）——加粗位置都在板块开头或转折处，承担"这一段我要说的就是这句"的功能
> - **4 连排比新模板**：「谁定义 eval，谁就在定义模型变强的方向。/ 谁定义 reward，谁就在定义 agent 学会什么。/ 谁控制数据闭环……/ 谁控制部署权限……」——主语反复 + 谓语递进，4 句独立成段，撑住骨架金句下面的论证
> - **"如果...就会..." 反馈链短句**：「如果反馈来自商业收入，AI 会往商业化走。/ 如果反馈来自用户时长，AI 会往黏人走。」5 连排穷举可能性，每句独立成段
> - **角色拆解排比**（工厂隐喻继续）：「有人负责数据。/ 有人负责 infra。/ 有人负责 eval。/ 有人负责 post-train……」7 连排把抽象组织拆成可数角色，与 5/13 "网络是神经系统" 同构
> - **结尾改造**：5/18 不用 2 词反差收束，落在 3 句判断收尾（「不是交给 AI。/ 是交给了一个由便利、懒惰、效率和默认设置组成的系统。」）——证明 Mode B 结尾仍有变体空间，不机械套 2 词反差
> - **英文术语裸奔可接受范围扩大**：alignment、eval、reward、post-train、long horizon、agent、system prompt 直接出现，仅 alignment 用括号补"中文经常翻译成对齐"。技术读者群已默认这些术语，过度翻译反而破坏节奏
>
> **2026-05-17 补充观察**（基于《5 把尺子量英伟达》）：
> - **“几把尺子”框架成型**：用 3-7 个维度量一个复杂对象，每个维度短段推进，最后承认答案拧巴。适合估值、AI 能力、项目判断这类无法一句话定性的主题
> - **先承认偷懒判断，再用数据打脸自己**：开头从“第一反应贴标签”写起，中段一把尺子一把尺子校正，结尾落在“先量一遍，再害怕”。这类自我纠偏比纯观点文更像作者本人
> - **“错问题”句式有效**：不是直接回答“贵不贵”，而是指出问题本身问错了，再拆成不同时间尺度和不同指标。可复用到 AI、投资、工具判断里
> - **大对象要落到可理解的比例**：比如“每 100 美元里差不多 8 美元自动流向英伟达”，把指数权重翻成生活化现金流，读者更容易跟上

> - **2 词反差收束作为新结尾模式连续出现**：「不是云。是厂。」「不是陪伴。是窗户。」（5/13、5/14 连续）——两个 1-2 字短句，前否定后肯定，独立成段。Mode B 长文新高频结尾式

> - **加粗骨架金句变密**：5/14 / 5/15 出现 2-3 个加粗判断式（**让 AI 适应姥姥。** / **贴图这个动作必须便宜到可以浪费。**）——加粗职能从"自我提醒"扩展到"骨架金句"作为板块锚点
> - **空 `##` 切板块继续巩固**（5/13、5/15）——长文不再倚赖小标题，纯用空 `##` 切节奏

>
> **2026-04-29 新增观察**：
> - 时事评论型（Mode B 新子类）：基于公共事件（DeFi 安全、OpenAI 官司），不依赖个人经历，但骨架仍是作者的判断。结构：事件起点 → 加粗的一句话核心判断 → 多角度展开 → 落到"对行业/我自己意味着什么"
> - 数据 + 观察混用：会先甩出 The Block / DefiLlama 这类来源数据，紧接着自己重新拉一遍数据反驳/补充原文（"我自己又顺手拉了一下"），形成"主流口径 vs 我的口径"的张力
> - 「明天继续。」作为新的固定收尾（《恢复日更》《DeFi 在救火》），可独立成段
> - 空 `##` 作为板块分隔（《OpenAI 的官司》），用纯视觉切分代替小标题，让节奏更慢更深
> - 行业类长文的金句更像"骨架金句"——加粗一句作为主论点（"DeFi 正在从蛮荒市场，走向准金融体系。""稳定币是支付和美元流动性叙事。"）

> - **工具实验日记可从一句操作判断打头**：不是介绍工具，而是先写"今天试了一个新玩法"，把实验感钉住，再展开具体工作方式

> - **工具文章仍然需要生活/市场碎片收尾**：AI 工作流之后接 VPS、旧电脑、链上机会，保持作者"一天里多条线同时推进"的真实节奏

> - **教程型长文可以从生活现场切入**：先写修车、孩子、家务，再自然转到期权学习。知识密度高，但入口仍是当天真实场景，不是"今天讲一个概念"

> - **工具评价可以从淘汰动作开头**：先写"今天彻底把 X 淘汰了"，不是先介绍新工具。真实替换动作比功能列表更像作者本人
> - **产品判断落在体感摩擦**：卡顿、等待、黑盒、反馈延迟这类细节比参数更有说服力。"会不会皱一下眉"这种判断句可以直接做核心论点
> - **AI agent 文章可用角色变化收束**：从"帮我写东西"到"帮我做事"再到"把电脑交给它"，用一句工作关系变化承接技术变化
> - **短句回扣旧梗有效**：旧文章里的"没有钱的小龙虾..."可以被新文章自然回收，形成连续叙事；但只回扣一句，不做长解释
>

> - **产业技术文可以从一个公司反推整套系统**：先写 Cerebras 的整片晶圆级芯片，再转向 NVIDIA、CUDA、国产替代。入口具体，落点是系统竞争
> - **空标题 `##` 适合承载长链条推演**：每个板块只推进一层判断，不需要解释性小标题，读者靠连续短段跟着走
> - **技术术语后面要立刻翻成人话**：CUDA、NVLink、HCCL、CANN 可以出现，但后面要接"出问题谁能修""客户换你以后麻烦更少"这种生活化判断
> - **产业结论要落在默认值竞争**：不是列参数，而是问谁能让客户少做选择、少救火、少迁移成本。这个判断适合 AI 基建、国产替代、数据中心建设类文章

## 一、声音——这个人说话是什么感觉

不是"风格规则"，是一个人的说话方式。写之前先把这个人装进脑子里：

- **像在微信群里跟几个老朋友聊今天发生的事**。不是写文章，不是做分享，就是聊天
- **想到哪说到哪**。话题之间不需要过渡，说完一件事，换个 `##` 就开始下一件
- **情绪是一闪而过的**，不会停下来分析自己的感受。"心里一慌"就完了，不会写"这件事让我意识到恐惧的本质"
- **观点直接甩出来当事实说**。"简直一无是处"，不是"我个人认为这可能不太好"
- **坦诚到近乎裸露**。会写对孩子发脾气、错过交易亏钱、安全事故、锻炼又断了。不粉饰
- **有时候碎，有时候厚**。一篇文章可以只有 130 字（《USD1 续杯》），也可以 1800 字（《请珍惜最后的捡钱时光》）。长短不是质量指标，是当天的状态
- **坦诚有两种用法**：日记体里的坦诚是实时的、不加工的；长文（Mode B/D）里的坦诚是策略性的——选已经翻篇的失败来展开，最敏感的部分留白（"细节不展开"）。后者不是虚伪，是对读者注意力的尊重

## 二、开头——55 篇文章的开头规律

### 绝对规律：第一句话永远是一个具体的事

从 55 篇文章中没有找到一篇以抽象观点、自我介绍或"今天我想聊聊"开头的。

### 四种开头类型（按频率排序）

**类型 1：事件陈述（最常见，~50%）**
直接说发生了什么，不加评价。
```
"今天 OPN 终于 TGE 了，初期价格在 0.45 左右波动，现在已经跌破了 0.4。"
"今天沪指17连阳，三市成交额超过3.6万亿。"
"今天恒生科技指数收盘跌了 2.91%。"
"OPN终于确定了3.5号TGE。"
"明天幼儿园开学，今天先把俩娃送到姥姥家了。"
```

**类型 2：个人状态/感受切入（~25%）**
从自己当下的状态开始，一句话建立情绪基调。
```
"最近睡眠有点少，所以做事说话都有点浮躁。"
"本命年了。"
"今天又是围绕着孩子打转的一天。"
"昨晚睡前喝了一瓶啤酒，感觉睡眠质量还不错。"
"养孩子很简单，但是负责任的养育孩子很难。"
```

**类型 3：他人动态/引用（~15%）**
转述别人说的话或做的事，作为话题引子。
```
"周一的时候，静姐给我发消息，问我今年的目标是否确定。"
"前几天看老华的分享'复利靠的是时间，再有50年，即便运气一般，也能踏入无人之地。'"
"今天最令人炸裂的消息，'炸鸡哥'跑路了。"
```

**类型 4：数据/价格切入（~10%）**
一个数字直接建立语境。
```
"0.33 开盘，最高到了 0.66，目前稳定在 0.6 以上。"
"今天见到了入圈以来的大饼最低价。"
"Claude code的每周限额是从周五开始重置，今天才周一我已经用了50%+了。"
```

### 绝对禁止的开头
```
❌ "今天是充实的一天"（太泛）
❌ "关于XX，我想分享一下"（太正式）
❌ "说到XX，很多人可能不知道"（说教）
❌ "在这个快速变化的时代"（空话）
❌ 任何形式的自我介绍或背景铺垫
```

## 三、结尾——散着走，不收

### 绝对规律：没有一篇文章以"总结全文"或"感谢阅读"结尾

### 五种结尾类型

**类型 1：下一步动作（最常见）**
说下一件要做的事，随意收掉。
```
"明天换成 KIMI 试试。"
"明天继续完善一下做市逻辑。"
"今天太晚了，明天好好复盘一下。"
"明天继续，目标是进入测试流程。"
```

**类型 2：生活碎片收尾**
用一句跟正文无关的生活细节结束，反差感。
```
"吃个布洛芬睡觉了。"
"今天吃到了好吃的铁板烧。"
"今天闺女继续输液，愁。"
"晚上陪老婆去看朱一龙，回家睡觉！"
```

**类型 3：金句/断言**
一句有力量的话，但不是总结全文，更像是贴在墙上的便签。
```
"安全不增加收益。它保证收益能留得住。"
"别等到有一天，你也收到那封邮件。"
"何其有幸。"
"好项目，烂结局。"
```

**类型 4：开放式留白**
问题抛出去，不给答案。
```
"不急着找到答案，先把问题看清楚。"
"今天这篇公众号文章就是用这套工作流自动生成、自动发送的，你能看出来吗？"
```

**类型 5：自我激励**
给自己打气，偏口语。
```
"代码是杠杆，扩张的边际成本是0，东边不亮西边亮，加油！"
"还有很多想法可以交给他实现，期待的搓手手。"
```

**类型 6：品牌标语（仅 Mode B/D 长文使用）**
用一句跨文章复用的签名式断言收束全文，让读者记住"这句话是你说的"。
```
示例（Mei 的用法）："执行力是普通人最顶级的认知，拥有坚持自律才是最牛逼的天赋。"
```
你自己的候选标语可以从已有金句中提炼——"安全不增加收益，它保证收益能留得住"这类已经有品牌潜力。不是每篇都要，但写长文时有意识地锤炼一句可以跨文章复用的话。

## 四、情绪表达——身体反应，不是心理分析

作者表达情绪的方式是**身体反应和动作**，不是形容词和分析。

### 真实用法（从文章中提取）

| 情绪 | 作者怎么写 | AI 容易写成 |
|------|-----------|------------|
| 紧张 | "心里一慌" "心里一紧" | "这让我感到非常紧张" |
| 恐惧 | "惊出我一身冷汗" | "这个发现令人担忧" |
| 震惊 | "一个激灵醒来" "着实把我震惊了" | "这个消息让我深感震惊" |
| 遗憾 | "留下了不争气的眼泪" | "这让我感到非常遗憾" |
| 疲惫 | "脑子要木掉了" "行尸走肉" "累的我够呛" | "今天感到非常疲惫" |
| 兴奋 | "搓手手" "日子突然有了奔头" | "这让我感到非常兴奋" |
| 愤怒 | "简直一无是处" | "我对此感到非常失望" |
| 释然 | "心里的一块大石头也落地了" | "终于松了一口气" |
| 忧愁 | "愁。"（一个字） | "这让我有些担忧" |

### 规律
- 身体 > 心理："心里一慌" > "感到焦虑"
- 动作 > 状态："一个激灵醒来" > "被吓到了"
- 短 > 长："愁。" > "心情沉重"
- 具体 > 抽象："脑子要木掉了" > "精神状态不佳"

反思：

补救措施：

## 六、结构——刻意的不均匀

### 板块结构

**日记体（~70%）的典型结构：**
- 2-4 个 `##` 板块，每个板块一个独立话题
- 板块之间没有逻辑关系，不需要过渡
- 话题随意混搭：市场 + 家庭 + 项目 + 锻炼

**板块长度必须不均匀：**
```
✅ 主话题 8 段 + 次话题 2 段 + 锻炼 1 句
✅ 话题A 3 段 + 话题B 3 段 + 话题C 1 句
✅ 全文无 ## 标题，纯流水式（如《姗姗来迟的OPN》《元宵节快乐》）

❌ 话题A 3 段 + 话题B 3 段 + 话题C 3 段（太均匀）
```

**特殊板块模式：**
- "锻炼"板块：出现在约 40% 的文章末尾，极短。格式为打卡清单或"今日无。"
- "明日交付"板块：早期文章常见，后期消失。1-3 条任务清单
- 菜单/食物列表：做饭相关文章会列出所有菜名，逐行排列

### 段落节奏

- **大部分段落 1-2 句**
- **偶尔一个长段落**（讲一件完整的事，如 ETH 做市那段）
- **允许一个词/句独立成段**："很神奇。" "愁。" "本命年了。"
- **短句和长句交替**，不要全是短句也不要全是长句
- **有时候整篇文章没有 ## 标题**，就是一段接一段写下来（速记型/情绪低沉时）

### 数字的说法
- "96k" 不是 "96,000美元"
- "一百六十多刀" 不是 "$161.89"
- "30 分" 不是 "30 积分"
- "六百多刀" 不是 "$640"
- 大饼、油 优先于 BTC、USDT

## 七、风格模式分类

### 模式 A：日记体（~70%，最常用）

**触发条件**：日志内容碎片化，多个不相关话题

**子类型：**

| 子类型 | 字数 | 特征 | 代表作 |
|--------|------|------|--------|
| 速记型 | 130-250 | 极短段落，纯事实，可能无 ## 标题，情绪低或太忙 | 《USD1 续杯》《赛道没了》《被卷的焦虑》《行情继续低迷》 |
| 标准型 | 250-500 | 2-4 个 ## 板块，某板块稍微展开 | 《舍得》《修心》《纪念日》《开工大吉》 |
| 厚实型 | 500-800 | 某板块有深度叙事，其他正常 | 《同路人与同心人》《放纵日为什么不焦虑》《尘埃落定》 |

**日记体的暗线——看起来随便聊，但有排列：**

1. **最有料的话题打头**。读者刷到推送只给3秒，第一段决定读不读下去。写完后扫一眼：最有意思的那段是不是在第一个？不是就挪上去
2. **主话题甩一句判断**。不是加分析，是把你脑子里已经想了的那句话写出来。你本来就习惯"观点直接当事实说"，只需要确保主话题上**说了**，不只是记录了事实。对比："OPN 跌到 0.3 了" vs "OPN 跌到 0.3 了，6500万融资开盘就破发六折，市场对这类项目的耐心耗尽了。"——后者只多了一句话
3. **碎话题碎到底**。碎话题的功能是节奏和人格，不是内容。一句话或一个词就够——"锻炼：今日无。""晚上做了个红烧肉，娃吃了两碗饭。"写到2-3段就不碎了，要么提成主话题，要么砍回一句
4. **写完回看：有没有一句陌生人也能带走的话**。私人日记写给自己，公众号日记写给读者。区别就在这里——每篇至少一句脱离上下文也成立的话。不用刻意，你已经在做了（"安全不增加收益""好项目，烂结局"），只需要写完后**确认它存在**

### 模式 B：深度思考（~15%）

**触发条件**：单一强事件/强情绪值得多角度展开

**特征：**
- 600-1500 字
- 3-5 个 `##` 段落，渐进式论证
- 从个人经历推导到普遍洞察
- 反问句做段落过渡
- 代表作：《web3 的信仰到底是什么？》《请珍惜最后的捡钱时光》《行情判断》《一个令人沮丧的消息》

**方法论包装（可选）：**
当文中出现 3 个以上相关论点时，考虑将它们打包为编号框架（3-7条）。读者会觉得拿到了"系统"而不只是观点碎片。打包信号：你发现自己在写"第一...第二...另外..."，这就是该打包的时候。不是每篇都要——没有素材硬凑框架比没有框架更差。

### 模式 C：框架/教程（~10%）

**触发条件**：系统方法论/工具教学

**特征：**
- 1000-2000 字
- 多级标题、表格、公式、编号步骤、代码块
- 语气相对正式，但仍有口语
- 代表作：《大杀器 - Claude Code Agent Teams》《我的第一个硬件钱包》《迟到的2025年总结（二）（三）》

### 模式 D：长叙事（~5%）

**触发条件**：回顾/总结/个人故事

**特征：**
- 1000-1800 字
- 时间线叙事，大量个人细节
- 情感浓度最高
- 代表作：《迟到的2025年总结》《成都之行复盘》《年》（童年回忆）

### 风格选择决策树

```
日志内容分析
├── 多个碎片话题，无突出主线 → 模式 A（日记体）
│   ├── 内容少/情绪低/太忙 → 速记型（130-250字）
│   ├── 正常一天 → 标准型（250-500字）
│   └── 某话题值得多写两段 → 厚实型（500-800字）
├── 单一强事件/强情绪 → 模式 B（深度思考）
├── 系统方法论/工具教学 → 模式 C（框架/教程）
└── 回顾/总结/个人故事 → 模式 D（长叙事）
```

## 十一、自检清单

写完文章后逐条检查，任何一条不通过就改：

1. 把文章念出来，**像不像在微信群里跟朋友说话**？
2. 有没有任何一句话是"AI 才会这么说"的？对照黑名单逐句检查
3. 板块长度是不是**不均匀**的？有没有碎的部分？
4. 开头第一句话是不是一个**具体事件**？
5. 结尾是不是**散着走**的，没有总结全文？
6. 情绪表达是**身体反应**还是**心理分析**？
7. 加粗是**自我提醒**还是**强调重点**？（后者要删）
8. 有没有在所有板块都"升华"？——有的板块就该是流水账
9. 文中有没有**至少一句可以被截图转发的话**？它是否足够独立、不依赖上下文就能成立？
10. 如果写了方法论/框架，条目数是否在**3-7之间**？（少于3不成体系，多于7读者记不住）
11. 文中提到了别人的帮助或观点吗？有没有**具名感谢**？（日记体随意，长文要有意识）
12. 日记体：**最有料的话题是不是在第一个位置**？碎话题有没有写超过一句？
