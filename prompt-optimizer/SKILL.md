# Prompt Optimizer

Analyze the user's question or goal, match it to the most effective prompting technique(s), and generate an optimized prompt ready to use.

## Core Principle

**表达清楚是基础，心法是催化剂。** 先确保问题本身表述清晰、上下文完整，再叠加心法激发 AI 最佳表现。

## Workflow

1. **Listen** - 理解用户想问什么、问谁、为什么问
2. **Clarify** - 如果用户需求模糊到无法判断心法，先问 1-2 个定位问题（"你想优化的是一个新 prompt 还是改进已有的？""你期望的输出是什么形式？"）。如果需求已经清晰，跳过此步
3. **Route** - 判断用户是"从零生成"还是"改进已有 prompt"：
   - **已有 prompt → 诊断模式**：分析现有 prompt 的弱点（上下文缺失？约束太松？格式没指定？角色不明？），针对性改进，不强套心法
   - **从零开始 → 生成模式**：用特征标记匹配心法（见下方）
4. **Optimize** - 基于心法模板或诊断结果生成优化后的完整 prompt
5. **Explain** - 简要说明为什么这样优化、prompt 中每部分的作用
6. **Deliver** - 给出可直接复制使用的 prompt（放在代码块中）
7. **Iterate** - 建议用户试用后反馈效果，如需要可继续迭代调整

## Technique Selection

为用户的问题打特征标签，根据标签组合推荐心法：

| 特征标签 | 推荐心法 |
|----------|----------|
| 需要专业视角 | T1 角色选定 |
| 信息量大/复杂 | T2 苏格拉底追问 |
| 有争议/需检验 | T3 AI辩论 |
| 有风险/需评估 | T4 预演失败 |
| 有成品范例 | T5 反向提示 |
| 要学习/理解 | T6 双层解释 |
| 需要特定输出格式 | T7 输出塑形 |
| 任务太大/多步骤 | T8 任务链拆解 |
| 标签均不匹配 | BASE 基础优化 |

多个标签可同时命中，参考"Combination Patterns"组合使用。

## BASE: 基础优化

当心法均不适用时，用以下清单补全 prompt：

- **角色**：你是谁？为什么问这个问题？（给 AI 提供回答的锚点）
- **目标**：期望的输出是什么？什么格式、什么深度、多长？
- **上下文**：相关背景信息、已有的尝试、已知的约束条件
- **示例**：如果有理想输出的样例，附上一个
- **边界**：明确不要什么（"不要泛泛的建议"、"不要超过 500 字"）

大多数 prompt 不需要心法，只需要把这五项补齐就能大幅提升质量。

## Eight Techniques

### T1: Role Selection (角色选定)

**When:** 需要专业领域建议，但不确定该找什么角色。

**Template:**
```
我想咨询关于 [领域/问题] 的建议。

请你先从该领域选择一位最适合回答这个问题的真实顶尖专家（历史或当代均可），说明你选择这个人的理由。

在开始前，你可以先问我 2 个定位问题，帮助你更准确地选择专家角色。

选定角色后，请以该专家的视角、思维方式和专业深度来回答我的问题。

我的问题是：[具体问题]
```

**Key:** 真实人物适合思维方式独特且广为人知的场景（"乔布斯谈产品"）；泛化角色适合需要特定技能组合的场景（"有10年经验的 SRE"）。优先选择你能说清"这个人会怎么想"的角色。

---

### T2: Socratic Questioning (苏格拉底追问)

**When:** 问题复杂、信息量大，直接问容易得到泛泛回答。日常使用频率最高。

**Template:**
```
我想请你帮我 [目标描述]。

但在给出方案之前，请你先通过追问来充分了解我的情况。规则如下：
- 每次只问我一个问题
- 根据我的回答继续追问
- 当你收集到以下信息后，给出完整方案：
  1) 我的具体场景和背景
  2) 我已尝试过的方法（如有）
  3) 成功的定义标准
  4) 主要约束条件（时间、资源、技术栈等）
- 如果某项信息对方案质量影响不大，可以跳过
- 给出方案时，说明你基于哪些关键信息做出的判断

我的初始背景：[简要背景]
```

**Key:** 用具体的信息清单作为停止条件，比置信百分比更可靠、更可控。

---

### T3: AI Debate (AI辩论)

**When:** 想检验观点、准备演讲/分享、对抗 AI 谄媚效应。

**Template:**
```
我有一个观点/方案，请你扮演一个强硬的反对者来与我辩论。

规则：
- 你的目标是找出我观点中的所有弱点
- 优先攻击最关键的假设，而非边缘细节
- 如果某个论点确实很强，承认它很强，然后去找下一个弱点
- 不要客气，不要顺着我说
- 如果我反驳了你的某个论点，立即换一个更强的角度继续挑战
- 直到你真的找不到可以反驳的角度，再告诉我你认为我的观点中哪些部分经受住了考验，哪些仍有隐患

我的观点是：[你的观点/方案]
```

**Key:** 适合演讲/分享前检验观点完善度，发现思维盲区。

---

### T4: Pre-mortem (预演失败)

**When:** 有具体计划/方案，需要提前发现风险和盲区。

**Template:**
```
以下是我的 [项目/计划/方案]：

[方案内容]

现在请你假设这个项目已经在 6 个月后彻底失败了。请以事后复盘的视角回答：

1. 最早出现的衰退信号是什么？出现在什么时间点？
2. 哪个决策是致命错误？当时为什么看起来是对的？
3. 有哪些被忽视的核心风险？
4. 如果重来，第一个要改的事情是什么？
5. 请基于真实的类似案例，写一篇简短的"失败复盘文章"

请尽可能具体，不要泛泛而谈。
```

**Key:** 人和 AI 都倾向乐观，这个心法强制切换到悲观视角。

---

### T5: Reverse Prompting (反向提示)

**When:** 有想要的成品范例，但不知道怎么描述需求。

**Template:**
```
这是我想要的成品范例：

[粘贴成品内容/上传图片]

请你：
1. 分析这个成品的结构、风格、节奏和关键特征
2. 倒推一个提示词，让我用它能稳定生成同风格同质量的内容
3. 逐句说明提示词中每一部分的作用
4. 指出哪些元素是风格核心（必须保留），哪些是可替换的变量
```

**Key:** 目的是学习结构和节奏规律，不是机械复制。支持文字和图片。

---

### T6: Dual-Layer Explanation (双层解释)

**When:** 想学习/理解一个概念，特别是技术性强的内容。

**Template:**
```
请用两个版本来解释 [概念名称]：

**版本一：大白话版**
用最生活化、最通俗的方式解释，就像在给一个没有任何专业背景但很聪明的人聊天。用生活中的比喻和例子。

**版本二：专业版**
用该领域的专业术语和框架来解释，面向有一定基础的从业者。包含关键术语的精确定义。

最后，列出两个版本之间的"认知差距"——从版本一到版本二，需要额外理解哪些关键概念？
```

**Key:** 通俗版追求生活化而非幼稚化（可以用"洗脚城大爷能听懂"作为心理标尺）。对照学习比单一版本更利于深入理解。

---

### T7: Output Shaping (输出塑形)

**When:** 知道要问什么，但需要控制 AI 的输出格式、结构或长度。

这不是一个独立模板，而是叠加在其他心法或基础优化之上的约束层。在生成 prompt 时，检查用户是否有以下需求并补充：

- **格式指定**：表格、编号列表、JSON、代码、Markdown、对话体
- **长度控制**："用 3 句话概括"、"不超过 500 字"、"写一篇 2000 字的长文"
- **结构约束**："先给结论，再展开论证"、"按优先级排序"、"分 pros/cons 两栏"
- **语气/风格**："正式书面语"、"口语化"、"技术文档风格"
- **排除项**："不要用比喻"、"不要列优缺点"、"跳过基础概念"

**Key:** 大量 prompt 效果差不是因为问错了问题，而是没告诉 AI 答案该长什么样。

---

### T8: Chain Decomposition (任务链拆解)

**When:** 用户扔来一个大任务（"帮我写商业计划"、"帮我做竞品分析"），直接执行会导致输出质量低或遗漏。

**Template:**
```
我需要完成这个任务：[任务描述]

请你不要直接开始做。先帮我把这个任务拆解成 3-7 个有序的子步骤，每个步骤：
1. 明确的输入和输出
2. 完成标准（怎样算做好了）
3. 预估的交互轮次（1轮还是需要追问）

拆解完成后，我们按顺序逐步执行。每完成一步，确认结果再进入下一步。
```

**Key:** 拆解后的每一步可以单独套用其他心法（如 T1 选角色、T2 追问细节），质量远高于一步到位。

## Combination Patterns

| Combo | When | Flow |
|-------|------|------|
| T1 + T2 | 复杂专业咨询 | 先选角色，再以该角色身份追问 |
| T2 + T4 | 方案制定 | 先追问了解全貌，再预演失败 |
| T5 + T1 | 复刻专业内容 | 先反向提示得到 prompt，再加角色提升专业度 |
| T3 + T4 | 重大决策 | 先辩论检验观点，再预演失败排查风险 |
| T6 + T2 | 深度学习 | 先双层解释入门，再追问深挖不懂的点 |
| T8 + 任意 | 大型任务 | 先拆解步骤，每步按需套用其他心法 |
| 任意 + T7 | 格式敏感 | 任何心法都可叠加输出塑形约束 |

## Output Format

When delivering optimized prompts to the user:

1. State which technique(s) you chose and why (1-2 sentences)
2. If in diagnostic mode, first list the existing prompt's weaknesses before presenting the improved version
3. Present the optimized prompt in a fenced code block, ready to copy
4. Briefly note any customization points the user might want to adjust
